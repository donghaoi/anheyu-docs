## Redis 可选说明

<Callout type="info" title="Redis 是可选的">
Anheyu 支持 **Redis 自动降级功能**，所有安装方式中 Redis 都是可选的：

- ✅ **使用 Redis（推荐生产环境）**：获得更好的搜索性能、精确的访客统计、支持分布式部署
- ✅ **不使用 Redis**：自动降级到内存缓存，所有核心功能正常使用，适合单机部署和小型应用

**如何选择？**

- 小型站点、测试环境：不需要配置 Redis
- 生产环境、大流量站点：推荐配置 Redis 无论选择哪种方式，Anheyu 都能正常运行！

</Callout>

## 安装

<Tabs items={['Docker Compose', 'Docker', '源码部署', '二进制文件']}>
  <Tab value="Docker Compose">
    ### Docker Compose 部署 (推荐)

    <Callout type="info" title="Docker Compose 包含的完整服务">
      使用 Docker Compose 启动时，将自动创建并启用以下完整的服务栈：

      **核心服务：**
      - 🚀 **Anheyu 主程序** - Go 后端 API + 内置 Next.js 前端（SSR），单容器提供完整 Web 服务
      - 💾 **Redis（可选）** - 缓存服务，提升系统性能。如不配置会自动降级到内存缓存
      - 🗄️ **PostgreSQL 数据库** - 主数据存储，支持复杂查询和事务

      **多媒体处理组件：**
      - 🖼️ **VIPS** - 高性能图像处理库，支持各种图片格式转换和缩略图生成
      - 🎥 **FFMpeg** - 强大的视频处理工具，支持视频格式转换、压缩和转码

      **安全特性（v1.8.0+）：**
      - 🔒 容器内应用以非 root 用户运行，通过 `su-exec` 自动降权
      - 📁 数据卷权限由 entrypoint 自动处理，用户无需手动设置

      这些组件协同工作，为您提供完整的文件存储、处理和管理解决方案。无需手动安装和配置这些复杂的依赖，一键启动即可获得全功能的服务。
    </Callout>

    1. 前置准备，请参考 [Docker Compose 安装文档](https://docs.docker.com/compose/install/) 安装 Docker 和 Docker Compose，确保你的环境有 Docker 和 Docker Compose。<br/>
    你可以理解为 Docker编排，即提前编排好需要的内容，写在同一个配置文件中，然后通过一个命令启动，使用者只需要关心这个配置文件即可。

    2. 准备 `docker-compose.yml` 文件
    创建一个目录作为 Docker Compose 文件的存储目录，比如：

    ```bash
    mkdir -p ~/anheyu
    cd ~/anheyu
    ```

    将获取到的 docker-compose.yml 文件保存到此目录。
    <Tabs items={['社区版', 'Pro 版']}>
      <Tab value="社区版">
        如下是一个包含所有必要服务的社区版 docker-compose.yml 文件示例。将其保存到服务端。

        ```yaml
        services:
          anheyu:
            image: anheyu/anheyu-backend:latest
            container_name: anheyu-backend
            depends_on:
              - anheyu_postgresql
              # Redis 是可选的，如果不需要可以注释掉下面这行
              - anheyu_redis
            restart: always
            ports:
              - 8091:8091
            environment:
              - ANHEYU_DATABASE_TYPE=postgres
              - ANHEYU_DATABASE_HOST=anheyu_postgresql
              - ANHEYU_DATABASE_USER=anheyu
              - ANHEYU_DATABASE_NAME=anheyu
              - ANHEYU_DATABASE_PORT=5432
              - ANHEYU_DATABASE_PASSWORD=${ANHEYU_DATABASE_PASSWORD:-changeme}
              # Redis 配置（可选）
              # 如果想使用 Redis，保持下面两行；如果不想使用，注释掉即可
              # 不配置时应用会自动降级到内存缓存
              - ANHEYU_REDIS_ADDR=anheyu_redis:6379
              - ANHEYU_REDIS_PASSWORD=${ANHEYU_REDIS_PASSWORD:-changeme}
              # 前端配置（可选，v1.8.0+）
              # 默认使用内置 Next.js 前端，无需配置
              # 如需指向独立部署的前端服务，取消注释下面这行
              # - ANHEYU_FRONTEND_URL=http://your-frontend:3000
            volumes:
              - ./data:/anheyu/data
              - ./themes:/anheyu/themes
              - ./static:/anheyu/static
              - ./backup:/anheyu/backup

          anheyu_postgresql:
            image: postgres:17
            container_name: anheyu_postgresql
            expose:
              - "5432"
            environment:
              - POSTGRES_USER=anheyu
              - POSTGRES_DB=anheyu
              - POSTGRES_PASSWORD=${ANHEYU_DATABASE_PASSWORD:-changeme}
            volumes:
              - database_postgres:/var/lib/postgresql/data

          # Redis 服务（可选）
          # 如果不需要 Redis，可以注释掉整个 anheyu_redis 服务
          anheyu_redis:
            image: redis:latest
            container_name: anheyu_redis
            command: >
              redis-server --requirepass ${ANHEYU_REDIS_PASSWORD:-changeme}
            expose:
              - "6379"
            volumes:
              - redis_data:/data

        volumes:
          database_postgres:
          redis_data:
        ```

        <Callout type="info" title="密码配置说明">
          上面的配置使用了环境变量替换（`${VAR:-default}`），默认密码为 `changeme`。

          **生产环境请务必修改密码：**
          - 方式 1：创建 `.env` 文件（推荐），在 `docker-compose.yml` 同级目录下创建：
          ```bash
          ANHEYU_DATABASE_PASSWORD=你的数据库密码
          ANHEYU_REDIS_PASSWORD=你的Redis密码
          ```
          - 方式 2：在启动命令中指定：`ANHEYU_DATABASE_PASSWORD=xxx docker compose up -d`
        </Callout>
      </Tab>
      <Tab value="Pro 版">
        如下是一个包含所有必要服务的 Pro 版 docker-compose.yml 文件示例。将其保存到服务端。

        ```yaml
        # 内置模式（推荐）: Go + Next.js 单容器，只暴露 8091 即可
        services:
          anheyu:
            image: anheyu/pro:latest
            container_name: anheyu-backend
            depends_on:
              anheyu_postgresql:
                condition: service_healthy
              anheyu_redis:
                condition: service_healthy
            restart: always
            ports:
              - 8091:8091
            environment:
              - ANHEYU_DATABASE_TYPE=postgres
              - ANHEYU_DATABASE_HOST=anheyu_postgresql
              - ANHEYU_DATABASE_USER=anheyu
              - ANHEYU_DATABASE_NAME=anheyu
              - ANHEYU_DATABASE_PORT=5432
              - ANHEYU_DATABASE_PASSWORD=${ANHEYU_DATABASE_PASSWORD:-}
              - ANHEYU_REDIS_ADDR=anheyu_redis:6379
              - ANHEYU_REDIS_DB=0
              - ANHEYU_LICENSE_KEY=${ANHEYU_LICENSE_KEY}
              # 不设置 ANHEYU_FRONTEND_URL = 使用内置 Next.js（推荐）
              # 若用独立前端容器可设: ANHEYU_FRONTEND_URL=http://anheyu-frontend:3000
              # - ANHEYU_FRONTEND_URL=${ANHEYU_FRONTEND_URL:-}
            volumes:
              - ./data:/anheyu/data
              - ./themes:/anheyu/themes
              - ./static:/anheyu/static
              - ./backup:/anheyu/backup

          anheyu_postgresql:
            image: pgvector/pgvector:pg18
            container_name: anheyu_postgresql
            ports:
              - "5432:5432"
            environment:
              - POSTGRES_USER=anheyu
              - POSTGRES_DB=anheyu
              # 无密码时用 trust；线上建议设 POSTGRES_PASSWORD 并改为 scram-sha-256
              - POSTGRES_PASSWORD=${POSTGRES_PASSWORD:-}
              - POSTGRES_HOST_AUTH_METHOD=trust
            # PostgreSQL 18+ 镜像：卷须挂到 /var/lib/postgresql（不要挂到 .../data），
            # 否则启动会报 “pg_upgrade” / unused mount 类错误（见 docker-library/postgres #1259）
            volumes:
              - database_postgres:/var/lib/postgresql
            healthcheck:
              test: ["CMD-SHELL", "pg_isready -U anheyu -d anheyu"]
              interval: 5s
              timeout: 5s
              retries: 5

          # Redis 服务（可选，不需要可注释掉）
          anheyu_redis:
            # RDB v12 需要 Redis 7.4+，使用 7.2 会报 Can't handle RDB format version 12
            image: redis:7.4-alpine
            container_name: anheyu_redis
            command: redis-server --appendonly yes
            volumes:
              - redis_data:/data
            healthcheck:
              test: ["CMD", "redis-cli", "ping"]
              interval: 5s
              timeout: 5s
              retries: 5

        volumes:
          database_postgres:
          redis_data:
        ```

        <Callout type="warn" title="若曾用旧示例挂过卷（仅 Pro / PostgreSQL 18）">
          若你之前把 `database_postgres` 挂在 `/var/lib/postgresql/data` 且容器起不来：**空库/可丢弃数据**时执行 `docker compose down -v` 后改用上面挂载路径再 `up`；**已有生产数据**请先备份，再按 [PostgreSQL 大版本升级](https://www.postgresql.org/docs/current/upgrading.html) 或官方镜像说明迁移，不要直接删卷。
        </Callout>

        <Callout type="info" title="授权密钥配置">
          在 `docker-compose.yml` 同级目录下创建 `.env` 文件，填入授权密钥：
          ```bash
          ANHEYU_LICENSE_KEY=你的授权密钥
          ```
          Docker Compose 会自动读取 `.env` 文件中的环境变量。
        </Callout>

    <Callout type="info" title="v1.8.0+ Docker 架构说明">
      从 v1.8.0 开始，Docker 镜像采用新架构：

      - **前端变更**：前端由 Vue SPA 升级为 **Next.js SSR**，Go 后端内置反向代理自动将前台请求转发到 Next.js 服务
      - **端口说明**：用户只需暴露 **8091** 端口即可，内部 Next.js 服务（端口 3000）由 Go 后端自动代理，无需额外映射
      - **安全增强**：容器使用 `su-exec` 以非 root 用户运行应用进程，数据卷权限由 entrypoint 自动处理
      - **前端可选配置**：可通过 `ANHEYU_FRONTEND_URL` 环境变量指向外部 Next.js 服务，实现前后端分离部署

      **从旧版本升级的用户**：普通应用升级只需拉取最新镜像并重启；若同时升级 PostgreSQL 大版本，必须先备份并按数据库升级流程处理数据卷。

## 宝塔面板 WAF 配置

<Callout type="warn" title="使用宝塔面板的用户请注意">
  如果你使用宝塔面板部署 Anheyu，并且开启了 WAF（Web 应用防火墙），可能会导致部分 API 请求被拦截。你需要在 WAF 中将以下
  URL 规则加入白名单。
</Callout>

在宝塔面板中，进入 **网站** → **设置** → **WAF** → **URL 白名单**，添加以下规则：

```text
^/api/pro/ai-podcast/cache-stat
^/api/pro/ai-podcast/pre-cache
^/api/pro/articles
^/api/pro/article/ai-summary
^/api/pages/[1-9999]
^/api/pro/articles/
^/api/settings/update
^/api/pro/ai-podcast/stream-mixed
```

<Callout type="info" title="说明">
  这些 API 端点用于以下功能：
  - **ai-podcast**: AI 播客相关功能
  - **articles**: 文章管理和查询
  - **ai-summary**: AI 文章摘要生成
  - **pages**: 分页查询
  - **settings/update**: 系统设置更新

如果不添加白名单，这些功能可能会因为 WAF 拦截而无法正常使用。

</Callout>

## 版本号释义

本项目的版本号为 X.Y.Z 格式，但与常见的[语义化版本号规范](https://semver.org/lang/zh-CN/)有部分区别，具体释义如下：

X: 产品层面的重新设计，包含重大框架重构，会涉及大范围功能变更与配置变更，更新前必须阅读相关文档

Y: 包含中大型新功能，及无法向下兼容的功能变更与配置变更，更新该版本号可能会需要修改配置或者移除一些原功能

Z: 不仅包含 BUG 修复、小型新功能，还会包含可以向下兼容的原功能更新与配置变更，原则上更新该版本号无需额外动作
